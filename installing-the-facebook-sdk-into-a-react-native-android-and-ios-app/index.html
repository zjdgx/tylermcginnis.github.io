How to Install the Facebook SDK into a React Native Android or iOS App

Aug 11, 2016

So I made this snarky tweet at 4 a.m. This is my peace offering.

"I have not failed. I have just found 10,000 ways to incorrectly install the Facebook SDK into a React Native app." - Thomas Edison

If you‚Äôre a native Android or iOS developer and are already familiar with their respective ecosystems, this blog post probably won‚Äôt do you much good.

But if you‚Äôre a JavaScript developer who doesn‚Äôt know what an Info.plist or a strings.xml file is ‚Äî and you‚Äôd sooner wait 45 minutes for npm install to finish than open up Xcode‚Äî this guide is for you.

Note that this is a just an excerpt from my much more comprehensive React Training‚Äôs React Native course.

Let‚Äôs assume you‚Äôre reading this because you‚Äôre more concerned with ‚Äúhow‚Äù than ‚Äúwhy.‚Äù So, unlike my typical posts, this one will be heavy on implementation details and light on understanding the abstractions.

I‚Äôm also assuming you‚Äôre using React Native .29 or above. If you‚Äôre using a version older than that, well, you have bigger problems than getting this tutorial to work.

We‚Äôre going to use Facebook‚Äôs react-native-fbsdk library, which is just a wrapper around the Facebook SDKs for Android and iOS. It will give us access to Facebook login, sharing, graph requests, and app events.

In your terminal run:

npm install --save

Next up, we need to do some linking.

Again, in your terminal run:

react-native link

Now let‚Äôs register a new app with Facebook.

Head over to Facebook‚Äôs developers site and ‚ÄúAdd a New App‚Äù.

When you see this screen select iOS‚Ä¶

‚Ä¶type in the name of your project, then hit the main submit button which at the time of this writing says ‚ÄúCreate new Facebook App Id.‚Äù Be sure to enter in a contact email address and select a Category.

If you‚Äôre lucky, at this point you‚Äôll get a fun little game about matching Tigers or watches. Facebook is just full of fun micro experiences üôÉ.

Once at this view, you‚Äôre going to do exactly what is says, and download the SDK to your ~/Documents folder.

Coming from the web, this is going to feel super strange, but it‚Äôs the only way I could get the SDK tied up properly without having to resort to using Pods.

At this point you should have successfully installed ‚Äúreact-native-fbsdk‚Äù and created a new iOS Facebook app. The FacebookSDK should be located in the ~/Documents folder of your computer.

Now for the moment we‚Äôve been training for.

Go to your project and double click on YourAppName.xcodeproj project to open it up in Xcode.

Once you have your project opened in Xcode, right click on your project‚Äôs name in the left sidebar and select ‚ÄúNew Group‚Äù and type in ‚ÄúFrameworks‚Äù.

Then open up your ~Documents/FacebookSDK folder from earlier and drag FBSDKCoreKit.Framework, FBSDKLoginKit.Framework, FBSDKShareKit.Framework into the Frameworks group you just created. The end result should look like this,

Now, click on the ‚ÄúBuild Settings‚Äù tag (pictured below) then search for ‚ÄúFramework Search Paths‚Äù. Once you find it add ‚Äú~/Documents/FacebookSDK‚Äù. (To get the little popup box double click on the highlighted line right below the word ‚ÄúSpero‚Äù then click the + button to add the text).

Now we need to dive into our Info.plist. For this step we‚Äôll need our Facebook App ID number. To get this, head back over to your Facebook developer dashboard and it will be on the home screen under ‚ÄúApp ID‚Äù.

Now open up your app in your preferred IDE and then open up the ‚ÄúInfo.plist‚Äù file located in ‚Äúios/YourAppName‚Äù

Now, right below this strange line

<string>????</string>

add this code,

<key>CFBundleURLTypes</key>
 <array>
  <dict>
   <key>CFBundleURLSchemes</key>
   <array>
    <string>fbYOUR-APP-ID</string>
   </array>
  </dict>
 </array>
 <key>FacebookAppID</key>
 <string>YOUR-APP-ID</string>
 <key>FacebookDisplayName</key>
 <string>YOUR-FACEBOOK-DISPLAY-NAME</string>
 <key>LSApplicationQueriesSchemes</key>
 <array>
  <string>fbapi</string>
  <string>fb-messenger-api</string>
  <string>fbauth2</string>
  <string>fbshareextension</string>
 </array>

Be sure to swap out ‚ÄúYOUR-APP-ID‚Äù with your Facebook APP Id and ‚ÄúYOUR-FACEBOOK-DISPLAY-NAME‚Äù with, well, your Facebook app display name.

Now head back to Xcode and grab your ‚ÄúBundle Identifier‚Äù number located in the ‚ÄúGeneral‚Äù tab.

Once you have that, copy it to your clipboard.

Now head over to your Facebook Developer dashboard again and click on ‚ÄúSettings‚Äù -> ‚ÄúBasic‚Äù -> ‚Äú+Add Platform‚Äù and select ‚ÄúiOS‚Äù.

Once you select iOS you‚Äôll see a place to enter in your ‚ÄúBundle ID‚Äù you just got from Xcode. Do that and be sure to select ‚ÄúSave Changes‚Äù.

Now head back to your text editor and open up ‚Äúios/PROJECT-NAME/AppDelegate.m‚Äù

Right above @implementation AppDelegate you need to import FBSDKCoreKit. Go ahead and add

#import <FBSDKCoreKit/FBSDKCoreKit.h>

Now right above the ‚Äú@end‚Äù line (and after the didFinishLaunchingWithOptions block) add this code,

- (void)applicationDidBecomeActive:(UIApplication *)application {
  [FBSDKAppEvents activateApp];
}

- (BOOL)application:(UIApplication *)application
    openURL:(NSURL *)url
    sourceApplication:(NSString *)sourceApplication
    annotation:(id)annotation {
    return [[FBSDKApplicationDelegate sharedInstance] application:application
      openURL:url
      sourceApplication:sourceApplication
      annotation:annotation];
}

So your final AppDelegate.m file will look something like this,

#import "AppDelegate.h"
#import "RCTBundleURLProvider.h"
#import "RCTRootView.h"
#import <FBSDKCoreKit/FBSDKCoreKit.h>

@implementation AppDelegate
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions
{
  NSURL *jsCodeLocation;
    jsCodeLocation = [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index.ios" fallbackResource:nil];
    RCTRootView *rootView = [[RCTRootView alloc] initWithBundleURL:jsCodeLocation
      moduleName:@"Spero"
      initialProperties:nil
      launchOptions:launchOptions];
    rootView.backgroundColor = [[UIColor alloc] initWithRed:1.0f green:1.0f blue:1.0f alpha:1];
    self.window = [[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds];
    UIViewController *rootViewController = [UIViewController new];
    rootViewController.view = rootView;
    self.window.rootViewController = rootViewController;
    [self.window makeKeyAndVisible];
    return YES;
}
- (void)applicationDidBecomeActive:(UIApplication *)application {
  [FBSDKAppEvents activateApp];
}
- (BOOL)application:(UIApplication *)application
            openURL:(NSURL *)url
  sourceApplication:(NSString *)sourceApplication
         annotation:(id)annotation {
  return [[FBSDKApplicationDelegate sharedInstance] application:application
      openURL:url
      sourceApplication:sourceApplication
      annotation:annotation];
}
 @end

Good news is that‚Äôs all you need to do for iOS. If you‚Äôre not building an Android app as well, you‚Äôre all set. However, if you are (or if you ever plan on) building an Android app as well, continue on.

Update: If at this point you‚Äôre still getting an error, it may be due to some recent changes in iOS 10 and Xcode 8. Head over to the ‚ÄúCapabilities‚Äù section in Xcode and enable ‚ÄúKeychain Sharing‚Äù

Now onto Android.

Now in your text editor head over to your ‚ÄúMainApplication.java‚Äù file located at android/app/src/main/java/com/

After all of the other imports in MainApplication.java, add the following imports.

import com.facebook.CallbackManager;
import com.facebook.FacebookSdk;
import com.facebook.reactnative.androidsdk.FBSDKPackage;

Now inside the MainApplication class, add the following properties,

private static CallbackManager mCallbackManager = CallbackManager.Factory.create();

protected static CallbackManager getCallbackManager() {
    return mCallbackManager;
}

Now go ahead and head to the bottom of your MainApplication class and add this Override to the class.

@Override
public void onCreate() {
  super.onCreate();
  FacebookSdk.sdkInitialize(getApplicationContext());
}

Now the last step in this file is you need to include the FBSDKPackage to your Array of React packages. Find the code that looks like this,

@Override
protected List<ReactPackage> getPackages() {
   return Arrays.<ReactPackage>asList(
    new MainReactPackage()
   );
}

and go ahead and add ‚Äúnew FBSDKPackage(mCallbackManager)‚Äù as a new list item on that array.

Once you‚Äôre finished, your code should look similar to this (changes are bold)

package com.spero;

  }

  }

    super.onCreate();
    FacebookSdk.sdkInitialize(getApplicationContext());
  }
}

Alright we‚Äôre really close. Now head over to *MainActivity.java (located at ‚Äúandroid/*app/src/main/java/com//‚Äù and do the following,

package com.spero;

Take a few deep breaths. We‚Äôre just a few changes away from Valhalla.

Head over to ‚Äústrings.xml‚Äù located at android/app/src/main/res/values/strings.xml and add a new string with your facebook app id.

Your strings.xml file will now look similar to this,
